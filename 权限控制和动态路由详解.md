## 📍 权限控制在哪里？

**位置**：`src/router/index.js` 第116-147行

### 权限控制的核心代码

```javascript
// 路由前置守卫 - 在路由跳转前检查权限
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem("token");

  if (token) {
    // 已登录
    const userInfo = JSON.parse(localStorage.getItem("userInfo"));

    if (userInfo.userType === 2) {
      // 管理员 (userType = 2)
      if (to.path.startsWith("/back")) {
        next();  // 允许访问后台
      } else {
        next("/back/dashboard");  // 强制跳到后台
      }
    } else if (userInfo.userType === 1) {
      // 普通用户 (userType = 1)
      if (to.path.startsWith("/back") || to.path.startsWith("/auth")) {
        next("/");  // 禁止访问后台和登录页
      } else {
        next();  // 允许访问前台
      }
    }
  } else {
    // 未登录
    if (to.path.startsWith("/back")) {
      next("/auth/login");  // 跳转到登录页
    } else {
      next();  // 允许访问公开页面
    }
  }
});
```

## 🎯 权限控制的工作流程

```
用户访问路由
    ↓
检查是否有token
    ↓
├─ 有token → 检查userType
│   ├─ userType = 2 (管理员)
│   │   ├─ 访问/back → ✅ 允许
│   │   └─ 访问其他 → ❌ 强制跳到/back/dashboard
│   │
│   └─ userType = 1 (普通用户)
│       ├─ 访问/back → ❌ 强制跳到/
│       ├─ 访问/auth → ❌ 强制跳到/
│       └─ 访问其他 → ✅ 允许
│
└─ 没有token
    ├─ 访问/back → ❌ 跳转到/auth/login
    └─ 访问其他 → ✅ 允许
```

## 🔐 权限控制的三个关键点

### 1️⃣ **Token验证**
```javascript
const token = localStorage.getItem("token");
if (token) {
  // 已登录
} else {
  // 未登录
}
```
- Token存储在localStorage中
- 每次路由跳转都检查token

### 2️⃣ **角色判断**
```javascript
const userInfo = JSON.parse(localStorage.getItem("userInfo"));
if (userInfo.userType === 2) {
  // 管理员
} else if (userInfo.userType === 1) {
  // 普通用户
}
```
- userType = 1：普通用户（前台）
- userType = 2：管理员（后台）

### 3️⃣ **路由拦截**
```javascript
if (to.path.startsWith("/back")) {
  // 检查是否有权限访问后台
}
```
- 根据路径前缀判断是否是后台路由
- 根据权限决定是否允许访问

## ❓ 什么叫动态路由？

### 定义
**动态路由** = 根据用户权限或数据，在运行时动态添加或隐藏路由

### 你的项目中的动态路由

#### 1️⃣ **动态参数路由**
```javascript
{
  path: "knowledge/article/:id",  // ← :id 是动态参数
  component: () => import("@/views/articleDetail.vue"),
  props: true,  // 将路由参数作为props传给组件
}
```

**使用示例**：
```javascript
// 点击文章时，动态跳转到不同的文章详情页
this.$router.push(`/knowledge/article/${articleId}`)

// 在articleDetail.vue中接收参数
const props = defineProps({
  id: String  // 接收路由参数
})
```

#### 2️⃣ **条件路由**（基于权限的动态显示）
```javascript
// 虽然代码中没有明确的动态添加路由，
// 但通过路由守卫实现了条件访问
if (userInfo.userType === 2) {
  // 管理员可以访问 /back 路由
  next();
} else {
  // 普通用户不能访问
  next("/");
}
```

## 🚀 如何改进权限控制？

### 现在的问题
- ❌ 权限控制比较简单，只有两个角色
- ❌ 没有细粒度权限控制（如按钮级别）
- ❌ 没有动态添加路由的功能

### 改进方案1️⃣：添加更多角色

```javascript
// 扩展权限类型
const rolePermissions = {
  1: ['/', '/consultation', '/emotion-diary', '/knowledge'],  // 普通用户
  2: ['/back/dashboard', '/back/knowledge', '/back/consultations', '/back/emotional'],  // 管理员
  3: ['/back/dashboard', '/back/knowledge'],  // 编辑（只能管理知识库）
  4: ['/back/consultations', '/back/emotional']  // 数据分析员
}

router.beforeEach((to, from, next) => {
  const token = localStorage.getItem("token");
  if (token) {
    const userInfo = JSON.parse(localStorage.getItem("userInfo"));
    const allowedRoutes = rolePermissions[userInfo.userType];

    if (allowedRoutes.some(route => to.path.startsWith(route))) {
      next();
    } else {
      next("/");
    }
  } else {
    if (to.path.startsWith("/back")) {
      next("/auth/login");
    } else {
      next();
    }
  }
});
```

### 改进方案2️⃣：动态添加路由

```javascript
// 根据权限动态添加路由
const addRoutesByRole = (userType) => {
  const roleRoutes = {
    1: [  // 普通用户路由
      { path: '/consultation', component: () => import('@/views/consultation.vue') }
    ],
    2: [  // 管理员路由
      { path: '/back/dashboard', component: () => import('@/views/dashboard.vue') },
      { path: '/back/knowledge', component: () => import('@/views/knowledge.vue') }
    ]
  }

  roleRoutes[userType]?.forEach(route => {
    router.addRoute(route)
  })
}

// 登录后调用
const handleLogin = async (credentials) => {
  const response = await login(credentials)
  localStorage.setItem('token', response.token)
  localStorage.setItem('userInfo', JSON.stringify(response.userInfo))

  // 动态添加路由
  addRoutesByRole(response.userInfo.userType)

  // 重新导航
  router.push('/back/dashboard')
}
```

### 改进方案3️⃣：按钮级别权限控制

```javascript
// 创建权限指令
const vPermission = {
  mounted(el, binding) {
    const userInfo = JSON.parse(localStorage.getItem('userInfo'))
    const requiredPermission = binding.value

    if (userInfo.permissions && !userInfo.permissions.includes(requiredPermission)) {
      el.style.display = 'none'  // 隐藏无权限的按钮
    }
  }
}

// 在组件中使用
<el-button v-permission="'delete_article'" @click="deleteArticle">
  删除
</el-button>
```

## 💡 面试时怎么讲

### 问：你的项目有权限控制吗？

**回答**：
> "有的。我们使用了基于角色的权限控制（RBAC）。
>
> 实现方式是在路由前置守卫中检查用户的token和userType：
> - 如果是管理员（userType=2），允许访问/back后台路由
> - 如果是普通用户（userType=1），只能访问前台路由
> - 如果没有token，访问后台时会跳转到登录页
>
> 这样就实现了基于角色的路由权限控制。"

### 问：什么叫动态路由？

**回答**：
> "动态路由有两种理解：
>
> 1. **动态参数路由**：路由中包含参数，如 /article/:id，可以根据不同的id访问不同的文章详情页
>
> 2. **条件路由**：根据用户权限或数据，在运行时动态决定是否允许访问某个路由。比如我们的项目中，管理员可以访问/back路由，但普通用户不能。
>
> 我们的项目中既有动态参数路由（/knowledge/article/:id），也有基于权限的条件路由。"

### 问：如何改进权限控制？

**回答**：
> "目前的权限控制比较简单，只有两个角色。可以改进的方向有：
>
> 1. **细粒度权限**：不仅控制路由级别，还要控制按钮级别（如删除、编辑按钮）
>
> 2. **动态添加路由**：根据用户权限，在登录后动态添加路由，而不是所有路由都写死
>
> 3. **权限缓存**：将用户权限缓存到Pinia，避免每次都从localStorage读取
>
> 4. **权限刷新**：支持权限动态更新，不需要重新登录"
