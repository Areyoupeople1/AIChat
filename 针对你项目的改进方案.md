## ❓ 你的问题解答

### Q1：Token刷新是前端的还是后端的？

**答案：两者都需要**

- **后端**：负责生成token、验证token、刷新token
- **前端**：负责存储token、发送token、处理token过期

#### 现在的流程
```
前端登录 → 后端返回token → 前端存储到localStorage
前端请求 → 添加token到请求头 → 后端验证token
后端返回401 → 前端跳转到登录页
```

#### 改进后的流程
```
前端登录 → 后端返回access_token + refresh_token
前端存储两个token

前端请求 → 添加access_token到请求头
后端返回401 → 前端调用刷新接口（用refresh_token）
后端返回新的access_token → 前端重发原请求
用户无感知
```

**你的项目现在没有token刷新，只需要改request.js的响应拦截器**

### Q2：这个状态怎么改？

**答案：修改响应拦截器**

#### 现在的代码（src/utils/request.js）
```javascript
service.interceptors.response.use(
  response => {
    const { data } = response
    if (data.code === '200') {
      return data.data
    } else if (data.code === '-1') {
      // Token过期 → 直接跳登录
      localStorage.removeItem('token')
      window.location.href = '/auth/login'
    }
  }
)
```

#### 改进后的代码
```javascript
let isRefreshing = false
let requests = []

service.interceptors.response.use(
  response => {
    const { data } = response
    if (data.code === '200') {
      return data.data
    } else if (data.code === '-1') {
      // Token过期 → 尝试刷新
      if (!isRefreshing) {
        isRefreshing = true
        const refreshToken = localStorage.getItem('refresh_token')

        // 调用后端刷新接口
        return axios.post('/api/user/refresh-token', { refreshToken })
          .then(res => {
            // 保存新token
            localStorage.setItem('token', res.data.data.token)
            localStorage.setItem('refresh_token', res.data.data.refresh_token)

            // 重发原请求
            return service(response.config)
          })
          .catch(() => {
            // 刷新失败 → 跳登录
            localStorage.removeItem('token')
            localStorage.removeItem('refresh_token')
            window.location.href = '/auth/login'
          })
          .finally(() => {
            isRefreshing = false
          })
      } else {
        // 正在刷新中 → 缓存请求
        return new Promise(resolve => {
          requests.push(() => {
            resolve(service(response.config))
          })
        })
      }
    }
  }
)
```

**关键点**：
- ✅ 前端检测401状态
- ✅ 调用后端刷新接口
- ✅ 保存新token
- ✅ 重发原请求

### Q4：怎么实现权限控制？

**答案：三层权限控制**

#### 第1层：路由级权限（已有）
```javascript
// src/router/index.js
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem("token")
  if (token) {
    const userInfo = JSON.parse(localStorage.getItem("userInfo"))
    if (userInfo.userType === 2) {
      // 管理员 → 允许访问 /back
      if (to.path.startsWith("/back")) {
        next()
      } else {
        next("/back/dashboard")
      }
    } else if (userInfo.userType === 1) {
      // 普通用户 → 禁止访问 /back
      if (to.path.startsWith("/back") || to.path.startsWith("/auth")) {
        next("/")
      } else {
        next()
      }
    }
  } else {
    if (to.path.startsWith("/back")) {
      next("/auth/login")
    } else {
      next()
    }
  }
})
```

#### 第2层：菜单级权限（新增）
```javascript
// src/components/Sidebar.vue
const visibleMenus = computed(() => {
  const allMenus = router.options.routes[0].children
  return allMenus.filter(item => {
    if (!item.meta?.roles) return true
    return item.meta.roles.includes(userType)
  })
})
```

#### 第3层：按钮级权限（可选）
