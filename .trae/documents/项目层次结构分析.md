# 心理健康AI助手项目层次结构分析

## 1. 项目整体架构

### 1.1 技术栈

* **前端框架**: Vue 3

* **构建工具**: Vite

* **UI框架**: Element Plus

* **状态管理**: Pinia

* **路由管理**: Vue Router

* **网络请求**: Axios

* **样式预处理器**: SCSS

* **图表库**: ECharts

* **富文本编辑器**: WangEditor

* **图标库**: Element Plus Icons

### 1.2 项目结构

```
├── public/              # 静态资源
├── src/                 # 源代码
│   ├── api/             # API接口定义
│   ├── assets/          # 静态资源
│   ├── components/      # 通用组件
│   ├── config/          # 配置文件
│   ├── router/          # 路由配置
│   ├── stores/          # 状态管理
│   ├── utils/           # 工具函数
│   ├── views/           # 页面组件
│   ├── App.vue          # 根组件
│   ├── main.js          # 入口文件
│   └── style.css        # 全局样式
├── 资料/                # 项目资料
├── package.json         # 项目配置
└── vite.config.js       # Vite配置
```

## 2. 核心功能模块

### 2.1 前台功能

* **首页**: 项目介绍和功能入口

* **AI咨询**: 与智能机器人进行对话

* **情绪日记**: 记录和管理情绪状态

* **知识库**: 浏览心理健康相关文章

* **文章详情**: 查看文章内容

### 2.2 后台功能

* **数据分析**: 查看系统数据统计

* **知识文章管理**: 管理知识库文章

* **咨询记录管理**: 查看用户咨询记录

* **情绪日志管理**: 查看用户情绪日志

## 3. 层次结构分析

### 3.1 表现层 (Views & Components)

* **页面组件** (`src/views/`): 包含所有页面级组件，如首页、咨询页面、情绪日记页面等

* **通用组件** (`src/components/`): 包含布局组件、导航组件、编辑器组件等可复用组件

### 3.2 业务逻辑层 (API & Utils)

* **API接口** (`src/api/`): 定义前端与后端的接口调用

* **工具函数** (`src/utils/`): 包含网络请求封装、工具方法等

### 3.3 状态管理层 (Stores)

* **状态管理** (`src/stores/`): 使用Pinia管理应用状态，如用户信息、后台数据等

### 3.4 路由层 (Router)

* **路由配置** (`src/router/`): 定义应用的路由结构，包括前台和后台路由

* **路由守卫**: 实现权限控制和导航逻辑

### 3.5 配置层 (Config)

* **配置文件** (`src/config/`): 存储应用配置信息

## 4. 数据流和交互逻辑

### 4.1 数据流

1. **用户操作** → **组件事件** → **API调用** → **后端响应** → **状态更新** → **UI更新**

### 4.2 权限控制

* **路由守卫**: 根据用户类型和登录状态控制页面访问权限

* **本地存储**: 使用localStorage存储token和用户信息

### 4.3 网络请求

* **请求拦截器**: 自动添加token到请求头

* **响应拦截器**: 统一处理响应数据和错误

## 5. 关键组件和页面

### 5.1 布局组件

* **FrontendLayout**: 前台布局，包含导航栏和页脚

* **BackendLayout**: 后台布局，包含侧边栏和顶部导航

* **AuthLayout**: 认证布局，用于登录和注册页面

### 5.2 核心页面

* **home.vue**: 前台首页，展示项目介绍和功能入口

* **consultation.vue**: AI咨询页面，实现与机器人的对话功能

* **emotionDiary.vue**: 情绪日记页面，记录用户情绪状态

* **frontendKnowledge.vue**: 前台知识库页面，展示文章列表

* **articleDetail.vue**: 文章详情页面，展示文章内容

* **dashboard.vue**: 后台数据分析页面

* **knowledge.vue**: 后台知识文章管理页面

* **consultations.vue**: 后台咨询记录管理页面

* **emotional.vue**: 后台情绪日志管理页面

* **login.vue**: 登录页面

* **register.vue**: 注册页面

## 6. API接口分析

### 6.1 前台API (`src/api/frontend.js`)

* 用户注册

* 情绪日记（添加情绪日记）

* 知识库（获取文章列表、获取文章详情）

### 6.2 后台API (`src/api/admin.js`)

* 用户管理

* 文章管理

* 咨询记录管理

* 情绪日志管理
